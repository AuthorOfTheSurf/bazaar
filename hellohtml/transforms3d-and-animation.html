Animations, Part 2: Defined Animations

Where transitions are implicit, meaning that they take place automatically when an element changes from one transitioned style to another, defined animations are explicit, meaning that you must state the animation steps that you would like to occur.

This animation approach affords you more flexibility, but as with many technologies, flexibility comes in exchange for additional complexity or planning.

To properly execute a defined animation, you must:

  1. Plan your animation out beforehand.
  2. Determine the keyframes of the animation.
  3. Define these keyframes in CSS.
  4. Give that particular sequence a unique name.
  5. Finally, state whether some element is supposed to use a particular animation.

The mechanics of defined animations consist of:

  • A @keyframes block that describes the “stops” in your animation. Each stop is defined by another { } block within the @keyframes block, prefixed the a percent value to indicate how far into the animation that stop should “take.” The @keyframes block itself must have a name, between single quotes ' '.
  • Once this animation sequence is defined, you can then apply it to any CSS selector using the animation-name property. Supply the desired @keyframes identifier and you’re done.
  • The specific application of the animation can then be customized through properties such as animation-duration, animation-iteration-count, and a few others. As usual, consult the [draft] specification for more options.
