CSS3 Transforms and Animations

Some of the most powerful features of the latest version of CSS can be found in its ability to transform and animate elements with very little code. You just state what you want, and an up-to-date, standards-compliant web browser will do the rest.

Transforms, Part 1

Simply put, you can rotate, scale, or translate (move) your elements with an immense degree of flexibility.

Just remember that these features are still very much bleeding edge, so their properties will need to be prefixed with –moz– and –webkit–.

You can also scale elements.

And you can combine different transforms, too.

Just make sure to mind your positions and sizes: elements occupy their untransformed areas on the page (note how the rotated element above this one is spilling over a bit).

You will also need to mind the transformation origin, or the reference point for the transformations. This defaults to the center of the element, but sometimes you will want to change it. Animations, Part 1: Transitions

When an element’s style changes from one to another, some style changes can be made smoothly.

Imagine if all your links behaved like this.

Effective transitions require some planning: you need to choose “start” and “stop” styles that go from one to the other in an appealing way (or not, if that is what you are after).

And yes, transitions do work with transformations (hold the mouse button over this paragraph).

Refer to the transition specification to see the full range of properties that can have animated transitions, plus additional options for customizing your transition effects.
